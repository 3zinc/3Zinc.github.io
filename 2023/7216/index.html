从石子合并看动态规划

PART 1：石子合并

石子合并这一题，算是典中典啦。这里给出题目大意：

有若干堆石子，每堆石子都有一个重量 wiw_iwi​（wi>0w_i>0wi​>0）。这些石子摆成一圈，每次可以合并相邻两堆石子（比如第 iii 堆和第 jjj 堆），合并的代价是 wi+wjw_i+w_jwi​+wj​，合并后的石子堆重 wi+wjw_i+w_jwi​+wj​。问经过 n−1n-1n−1 次合并，总代价最少是多少。

比如看一个例子，有 444 堆石头，w1∼w4w_1 \sim w_4w1​∼w4​ 本别是 4,9,5,44,9,5,44,9,5,4。一下是最优的合并方式：

  • 第一步：合并 w1w_1w1​ 与 w4w_4w4​，变成 333 堆石头 8,9,58,9,58,9,5，代价 888

  • 第二步：合并 w1w_1w1​ 与 w3w_3w3​，变成 222 堆石头 13,913,913,9，代价 131313。

  • 第三步，合并剩余两堆，代价 222222。

总代价是 22+13+8=4322+13+8=4322+13+8=43。

PART 2：算法一，记忆化搜索（分治）

对于这一个问题，我们可以思考使用分治来解决。

  • 原问题：将 i∼ji \sim ji∼j 的所有石子合并的最小代价。
  • 子问题一：将 i∼ki \sim ki∼k 的所有石子合并的最小代价。
  • 子问题二：将 k+1∼jk+1 \sim jk+1∼j 的所有石子合并的最小代价。
  • 基本情况：单独一堆石子，合并的代价是 000。
  • 合并：min⁡{\min\{min{ 子问题一 +++ 子问题二 }+i∼j\} + i \sim j}+i∼j 所有石子质量总和。

其中，i≤k≤j−1i \leq k \leq j-1i≤k≤j−1。

代码很简单。加上记忆化，易知其时间复杂度为 O(n3)\mathcal{O}(n^3)O(n3)。

PART 3：算法二，动态规划

我们发现，对于每一层 solve(i,j)，它都有某一个 solve(i,k),solve(k+1,j) 转移而来。所以我们可以考虑首先枚举区间长度，再枚举 iii，然后枚举 kkk 转移。

这一算法思想就是动态规划（DP）：

  • 状态：fi,jf_{i,j}fi,j​ 表示合并第 iii 堆到第 jjj 堆石子的最小代价。

  • 转移：fi,j=min⁡{fi,k+fk+1,j}+∑p=ijapf_{i,j}=\min\{f_{i,k}+f_{k+1,j}\}+\sum\limits_{p=i}^{j}{a_p}fi,j​=min{fi,k​+fk+1,j​}+p=i∑j​ap​

  • 初值：fi,j=∞(i≠j),fi,i=0f_{i,j}=\infty(i \ne j),f_{i,i}=0fi,j​=∞(i=j),fi,i​=0

  • 目标：min⁡fi,i+n−1(1≤i≤n)\min{f_{i,i+n-1}}(1 \leq i \leq n)minfi,i+n−1​(1≤i≤n)。

（再写动态规划的题的时候，建议写出如上转移方程，有助理解）

显然，对于枚举长度，需要 O(n)\mathcal{O}(n)O(n) 的时间；对于枚举 iii，需要 O(n)\mathcal{O}(n)O(n) 的时间；对于枚举 kkk 进行转移，也需要 O(n)\mathcal{O}(n)O(n) 的时间（求和可以用前缀和处理）。总时间复杂度为 O(n3)\mathcal{O}(n^3)O(n3)。（其实，如果仔细观察，你会发现动态规划与记忆化搜索（分治）有许多相似之处。其实，这两种方法只是动态规划的不同写法而已）

再整理一下，你可以发现动态规划需要满足的两个性质：

  • 最优子结构：其实从本质上来说，就是子问题的最优解，一定是原问题的最优解。比如说上面的分治过程中，solve(i,k)+solve(k+1,j) 最小的一个一定就是让 solve(i,j) 最小的子问题。这是最重要的一点。

  • 重叠子问题：也就是某一个同样的问题会被重复计算很多次。这在记忆化搜索中就非常明显。比如，solve(1,3) 可能被 solve(1,4) 调用，可能被 solve(1,5) 调用……但其实每次调用所得的结果是一样的。我们可以把这些答案记下来，只用计算一次。这样可以非常明显地优化算法。

PART 4：平行四边形优化

如果题目中，不是 n≤200n \leq 200n≤200 而是 n≤2000n \leq 2000n≤2000 呢？又该怎么办？

这是，我们要考虑对算法进行优化。

首先，我们要加入最优决策点这一概念。比如说 9,5,4,49,5,4,49,5,4,4 的最优决策点是 111，表示 solve(1,4) 是由 solve(1,1) 和 solve(2,4) 得来的。

然后思考：假设区间 (i,j)(i,j)(i,j) 的最优决策点是 k(i≤k<j)k(i \leq k < j)k(i≤k<j)，那么区间 (i−1,j)(i-1,j)(i−1,j) 的最优决策点可能在哪里呢？思考，可以得出 (i−1,j)(i-1,j)(i−1,j) 的最优决策点 ppp 绝对满足 p≤kp \leq kp≤k。

呢对于 (i,j+1)(i,j+1)(i,j+1) 区间呢？也可以得出，(i,j+1)(i,j+1)(i,j+1) 的最优决策点 qqq 绝对满足 q≥kq \geq kq≥k。

可能这一点优化看起来没什么了不起的，但是仔细一想，如果得出 (i,j)(i,j)(i,j) 的最优决策点是 kkk，那么 (i,j+1),(i,j+2),(i,j+3),⋯ ,(i,n)(i,j+1),(i,j+2),(i,j+3),\cdots,(i,n)(i,j+1),(i,j+2),(i,j+3),⋯,(i,n) 的最优决策点都 ≥k\geq k≥k。要知道，所有的最优决策点也只有 O(n)\mathcal{O}(n)O(n) 个，所以均摊下来，枚举 kkk 的时间复杂度约为 O(1)\mathcal{O}(1)O(1)。

如何运用上面的东西呢？对于 (i,j)(i,j)(i,j)，显然，已经计算完了 (i+1,j)(i+1,j)(i+1,j) 与 (i,j−1)(i,j-1)(i,j−1)。假设两个区间的最优决策点分别是 p,qp,qp,q，那么找 (i,j)(i,j)(i,j) 的最优决策点，就只用在 [q,p][q,p][q,p] 里找就行了。
